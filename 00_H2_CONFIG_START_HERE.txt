【步骤 2】在数据库工具中创建连接
──────────────────────────────────────────────────────────────────────────────
以 DBeaver 为例:
  1. Database → New Database Connection → H2
  2. URL: jdbc:h2:tcp://192.168.56.1:9092/file:./data/toolmanager;MODE=MySQL
  3. User: sa
  4. Password: (留空)
  5. Test Connection → Finish

更多工具的配置: 查看 H2_QUICK_REFERENCE.txt

【故障排查速查表】
================================================================================

问题: 无法连接
解决: 
  □ 检查应用是否启动: ps aux | grep java
  □ 检查 TCP 端口: telnet 192.168.56.1 9092
  □ 查看应用日志中是否有错误

问题: 拒绝访问 (401)
解决:
  □ 用户名: sa (小写)
  □ 密码: (完全留空，不输入任何字符)
  □ 在 H2 Web Console 验证: http://192.168.56.1:8080/h2-console

问题: 找不到数据库
解决:
  □ JDBC URL 必须正确: jdbc:h2:tcp://192.168.56.1:9092/file:./data/toolmanager;MODE=MySQL
  □ 数据库文件: ./data/toolmanager.mv.db
  □ 如不存在会自动创建

问题: 连接超时
解决:
  □ 确认网络连通: ping 192.168.56.1
  □ 确认应用仍在运行
  □ 检查防火墙是否开放 9092 端口

更多问题: 查看 H2_REMOTE_TCP_GUIDE.txt

【性能建议】
================================================================================

1. 使用 TCP 连接而不是频繁的 REST API 查询
2. 对于大数据量操作使用分页
3. 建立适当的数据库索引
4. 定期备份数据库文件: ./data/toolmanager.mv.db
5. 对于生产环境，迁移到 MySQL 或 PostgreSQL
