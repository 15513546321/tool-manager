# 文档预览优化方案

## 1. 安装现成的预览组件
```bash
npm install react-file-viewer react-markdown
```

## 2. 重新设计DocRepository.tsx布局
- 将版本历史从右侧边栏移到页面上方
- 采用水平卡片式布局展示版本
- 每个版本卡片包含：版本号、日期、文件名、查看、下载按钮
- 点击版本卡片切换预览内容

## 3. 替换自定义预览组件
- **PDF预览**：使用`react-file-viewer`替代自定义PDFViewer
- **Word/Excel预览**：使用`react-file-viewer`统一处理
- **Markdown预览**：使用`react-markdown`替代自定义MarkdownViewer
- **图片/代码预览**：保留现有ImageViewer和CodeViewer（工作良好）

## 4. 优化布局结构
```
┌─────────────────────────────────────────┐
│ 文档标题 + 描述 + 操作按钮              │
├─────────────────────────────────────────┤
│ 版本历史 (水平卡片布局)                │
│ [v1.0] [v0.9] [v0.8] ...          │
├─────────────────────────────────────────┤
│                                     │
│  文件预览区域 (独立滚动)               │
│                                     │
│                                     │
└─────────────────────────────────────────┘
```

## 5. 移除不必要的自定义组件
- 删除`components/PDFViewer.tsx`
- 删除`components/MarkdownViewer.tsx`
- 保留`components/ImageViewer.tsx`和`components/CodeViewer.tsx`

## 优势
- ✅ 解决版本历史被挤压的问题
- ✅ 使用成熟的组件避免PDF崩溃
- ✅ 减少维护成本，提高稳定性
- ✅ 统一的预览体验
